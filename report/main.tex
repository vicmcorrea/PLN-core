\section{Introdução}

Este projeto tem como objetivo classificar textos curtos em português brasileiro como positivos, negativos ou neutros. A entrada consiste em frases simples, como comentários de redes sociais, e a saída é um rótulo acompanhado de um escore numérico.

A análise de sentimentos é uma tarefa amplamente utilizada em aplicações como monitoramento de opinião pública, avaliação de produtos e apoio à tomada de decisão. Em ambientes digitais, há um grande volume de opiniões textuais produzidas continuamente, o que torna essa tarefa relevante tanto do ponto de vista prático quanto acadêmico.

No contexto do português brasileiro, essa tarefa apresenta desafios adicionais, como variações linguísticas, uso de gírias, ambiguidade semântica e forte dependência de contexto.

Entre os principais desafios estão:
\begin{itemize}
  \item dependência de contexto (ex.: ironia e sarcasmo)
  \item ambiguidade lexical
  \item impacto de negações e intensificadores
  \item cobertura limitada de léxicos
\end{itemize}

Além disso, trata-se de um problema didaticamente relevante em Processamento de Língua Natural, pois envolve a modelagem de fenômenos linguísticos complexos de forma estruturada.

Neste trabalho, adotamos uma abordagem simbólica baseada em léxico e regras explícitas, priorizando interpretabilidade em vez de aprendizado automático \citep{taboada2011lexicon,hutto2014vader,souza2012sentiment}.

\subsection{Recursos e ferramentas}

O sistema foi desenvolvido em Python e utiliza:
\begin{itemize}
  \item expressões regulares para normalização e tokenização
  \item modelo \texttt{pt\_core\_news\_sm} do spaCy para tokenização e lematização em português
  \item léxico OpLexicon v3.0 como recurso externo de polaridade
\end{itemize}

\subsection{Descrição dos processos}

\textbf{Escolha do léxico:}
O sistema pode utilizar um léxico manual reduzido ou o OpLexicon v3.0 como recurso externo. No caso do OpLexicon v3.0, os dados são carregados automaticamente e entradas duplicadas são agregadas por média de polaridade.

\textbf{Normalização:}
Consiste na remoção de URLs e menções, além da normalização de espaços e redução de caracteres repetidos.\\

\textbf{Tokenização e Lematização:}
O projeto mantém um tokenizador por expressões regulares para a versão simbólica simples, mas a configuração principal usada na CLI e no Streamlit utiliza o modelo \texttt{pt\_core\_news\_sm} do spaCy \citep{spacyModels}. Nessa configuração, o texto é tokenizado e cada item é convertido para sua forma lematizada, permitindo associar flexões como \texttt{gostei} e \texttt{gostaram} à forma \texttt{gostar} presente no léxico. Lemas auxiliares muito frequentes, como \texttt{ser} e \texttt{estar}, são filtrados para evitar correspondências espúrias no OpLexicon.\\

\textbf{Busca no léxico:}
Os lemas identificados no léxico recebem um escore base de polaridade.\\

\textbf{Regras simbólicas:}
As regras são aplicadas sequencialmente e incluem negação, intensificação, atenuação, contraste e tratamento de exclamações. Essas heurísticas são inspiradas em abordagens descritas na literatura \citep{hutto2014vader,kiritchenko2016effect}.\\

\textbf{Agregação e classificação:}
Os escores ajustados são somados e o resultado final é classificado como positivo, negativo ou neutro.

\section{Execução}

O repositório remoto do projeto é:

\url{https://github.com/vicmcorrea/PLN-core}

Para executar o projeto, é necessário preparar o ambiente virtual e instalar as dependências.

\subsection*{Execução via linha de comando}

Linux/MacOS:
\begin{verbatim}
./install.sh
source .venv/bin/activate
python main.py
\end{verbatim}

Windows:
\begin{verbatim}
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip setuptools
python -m pip install -e .
python main.py
\end{verbatim}

Também é possível analisar diretamente um texto:

\begin{verbatim}
python main.py "Eu amei o filme, foi muito bom!"
\end{verbatim}

O sistema permite dois modos:
\begin{itemize}
  \item análise de texto individual
  \item modo de comparação dos exemplos embutidos usando a mesma configuração principal do aplicativo
\end{itemize}

\subsection*{Recursos adicionais}

Na primeira execução, o léxico OpLexicon v3.0 é automaticamente baixado e armazenado localmente.

Além da interface em linha de comando, o projeto também oferece uma interface web simples utilizando Streamlit:

\begin{verbatim}
streamlit run streamlit_app.py
\end{verbatim}

ou, via opção acessível diretamente no navegador:

\url{https://pln-core.streamlit.app/}


A interface permite visualizar:
\begin{itemize}
  \item rótulo final
  \item escore
  \item texto normalizado
  \item lemas gerados para busca no léxico
  \item termos encontrados no léxico
  \item regras simbólicas aplicadas

\section{Recomendação de músicas}

Como extensão do sistema, foi implementado um módulo de recomendação de músicas baseado no sentimento detectado no texto. A recomendação ocorre após a classificação e utiliza um catálogo local com dez músicas, em que cada música possui um rótulo de sentimento e um valor de valência.

Dado um texto, o sistema filtra músicas com o mesmo rótulo e as ordena pela menor distância entre o escore do texto e a valência das músicas.

Valência (valence) representa o grau de positividade ou negatividade associado a uma música.

A recomendação segue a regra:

\begin{equation}
d(s) = |\,\text{valence}(s) - \text{score}_{texto}\,|
\end{equation}

A música recomendada é aquela com menor distância. O módulo é determinístico e não utiliza aprendizado de máquina ou APIs externas. Os valores de valência foram atribuídos manualmente com base na interpretação das letras, sendo uma aproximação simplificada.

\section{Conclusão}

O presente trabalho apresentou um sistema simbólico para análise de sentimentos em português brasileiro, baseado em léxico e regras explícitas, priorizando interpretabilidade ao longo de todo o pipeline.

Embora o sistema seja capaz de capturar padrões relevantes de polaridade em textos curtos, os resultados evidenciam limitações inerentes a essa abordagem. Em particular, a dependência da cobertura do léxico, a simplificação de fenômenos linguísticos complexos por meio de regras heurísticas e a análise predominantemente local do contexto impactam diretamente a qualidade das classificações. Mesmo com o uso de lematização, ainda há dificuldades na representação de nuances semânticas mais profundas.

Além disso, o módulo de recomendação, embora funcional, baseia-se em valores de valência atribuídos manualmente, o que limita a precisão das sugestões.

Essas limitações são esperadas em abordagens simbólicas e apontam caminhos naturais para evolução do sistema, como o enriquecimento do léxico, o refinamento das regras e a incorporação de técnicas mais robustas de modelagem linguística.

Em síntese, o projeto evidencia que, embora abordagens simbólicas ofereçam transparência e controle, sua efetividade depende fortemente da qualidade dos recursos e das simplificações adotadas, reforçando o papel dessas técnicas como um ponto de partida interpretável — mas não definitivo — para a análise de sentimentos.
